# Linting and Code Quality Rules

## Linting Tools
- Use language-specific linters (ESLint, golangci-lint, pylint)
- Configure linters in CI/CD pipeline
- Fix linting errors before committing
- Use auto-fix where possible

## Code Formatting
- Use formatters (Prettier, gofmt, black)
- Configure formatters consistently
- Format code on save
- Enforce formatting in CI

## Code Review
- Review all code changes
- Check for security vulnerabilities
- Verify test coverage
- Ensure code follows style guide

## Best Practices
- Keep functions small and focused
- Use meaningful variable names
- Remove dead code
- Document complex logic
